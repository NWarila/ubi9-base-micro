# NIST SP 800-190 Image Evidence

This repository emits a custom in-toto predicate for NIST SP 800-190 section 4.1 image countermeasures:

```text
https://nwarila.dev/attestations/nist-sp-800-190-image/v1
```

This is image-layer evidence for `ubi9-base-micro`. It is not CIS Docker evidence: CIS Docker Benchmark controls are host and daemon controls, while this predicate documents the image-control posture for this image and its published platform child digests.

PR builds generate and validate the predicate only. Push builds generate the predicate for each platform child digest, run the rootfs secret check first, attach the predicate with `cosign attest --type https://nwarila.dev/attestations/nist-sp-800-190-image/v1`, and verify it with the exact repository workflow identity.

## Control Mapping

| 4.1.1 Image vulnerabilities | addressed | Fixable HIGH and CRITICAL OS/library findings fail closed through both Trivy and Grype. | 4.1.3 Embedded malware | addressed | The image has a minimal rpmdb-enumerated package set and is scanned by Trivy and Grype over the published image contents. ## Predicate Contents

The predicate records:

- `predicateType`: the stable URI above.
- `standard`: NIST SP 800-190 section 4.1 scope and the not CIS Docker clarification.
- `subject`: image reference, platform, and architecture.
- `build`: source repository, revision, and digest-pinned UBI micro base image.
- `secretScan`: the passing rootfs secret-scan report path and scan counts.
- `controls`: the five section 4.1 entries with status, posture, and evidence pointers.
- `limitations`: bounded statements for CIS Docker, embedded-malware coverage, and architecture-scoped FIPS claims.

No control is marked not applicable in this repository's current mapping. If a future variant uses a not-applicable-with-reason status, the reason must be evidence-backed in the predicate and in this document.
